# Завдання 7. Аналітичний звіт для презентації

## Аналіз моделей машинного навчання для прогнозування COVID-19

### Процес підготовки даних

Підготовка даних була ключовим етапом у побудові ефективних моделей машинного навчання. Спочатку ми завантажили датасет COVID-19, який містив інформацію про випадки захворювання, смертність, вакцинацію та демографічні показники по різних країнах світу. Для цільової змінної регресії ми обрали `new_cases` (кількість нових випадків), а для класифікації створили бінарну змінну `high_cases`, яка вказує чи перевищує кількість нових випадків поріг у 1000.

Обробка категоріальних змінних включала One-Hot Encoding для континентів (`continent_Asia`, `continent_Europe`, `continent_North America`, `continent_Oceania`, `continent_South America`) та Label Encoding для коду країни (`iso_code`). Числові змінні (`total_cases`, `total_deaths`, `total_vaccinations`, `population`) були масштабовані за допомогою StandardScaler для забезпечення однакового масштабу ознак. Для класифікаційних моделей ми застосували SMOTE для балансування класів, оскільки спостерігався дисбаланс між позитивними та негативними прикладами.

### Навчання моделей та їх результати

Для регресії ми навчили чотири моделі: лінійну регресію, поліноміальну регресію (ступінь 2), Ridge та Lasso регресію. Результати показали, що всі моделі мають низьке значення R² (близько 0.01), що свідчить про слабку лінійну залежність між ознаками та кількістю нових випадків. MSE для всіх моделей був дуже високим (≈9.13×10⁹), що вказує на складність прогнозування нових випадків COVID-19 на основі наявних ознак.

Для класифікації ми порівняли логістичну регресію, дерево рішень, випадковий ліс та k-NN. Найкращу точність (Accuracy = 0.835) показав випадковий ліс, але з низьким показником повноти (Recall = 0.102). Логістична регресія мала найвищу повноту (Recall = 0.809), але низьку точність (Precision = 0.109). Дерево рішень показало найкращий баланс між точністю та повнотою (F1 = 0.183).

### Відбір ознак та покращення моделей

Використовуючи SelectKBest, ми визначили топ-10 найважливіших ознак, які включали `iso_code`, `total_cases`, `total_deaths`, `total_vaccinations`, `population` та бінарні ознаки континентів. Після навчання випадкового лісу лише на цих ознаках точність моделі покращилася з 0.835 до 0.855, що підтверджує ефективність відбору ознак.

### Висновки та рекомендації

Найкращою моделлю для класифікації виявився випадковий ліс з відібраними ознаками (Accuracy = 0.855), тоді як для регресії всі моделі показали схожі результати з низькою прогнозною здатністю. Це пояснюється високою волатильністю пандемії COVID-19 та впливом зовнішніх факторів, які не були включені в датасет.

Для подальшого вдосконалення рекомендується: (1) включити часові ознаки (seasonality, trends), (2) додати політичні та соціальні індикатори (stringency index), (3) застосувати більш складні моделі (XGBoost, нейронні мережі), (4) провести глибший аналіз викидів та аномалій у даних. Класифікаційний підхід виявився більш ефективним для цього датасету порівняно з регресією.
